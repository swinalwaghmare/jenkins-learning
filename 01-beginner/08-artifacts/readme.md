## 🚨 Errors Encountered

During the implementation of this pipeline, I ran into the following issues and resolved them:

1. **JUnit reports were not found during the `post` stage**

   * Although the tests executed successfully, Jenkins failed with:

     ```text
     No test report files were found. Configuration error?
     ```

   * The `junit` step was searching for reports in:

     ```text
     target/surefire-reports/*.xml
     ```

     However, the Maven project was located inside:

     ```text
     sample-apps/java-app/
     ```

     so the actual reports were generated at:

     ```text
     sample-apps/java-app/target/surefire-reports/
     ```

   * **Fix:** Updated the `junit` step to use the correct relative path or wrapped it inside a `dir('sample-apps/java-app')` block.

2. **Archived artifacts were not found**

   * Similar to the JUnit reports, Jenkins attempted to archive artifacts from:

     ```text
     target/*.jar
     ```

   * Since the JAR file was generated inside the project directory, Jenkins could not locate it.

   * **Fix:** Changed the archive path to:

     ```text
     sample-apps/java-app/target/*.jar
     ```

     or executed the `archiveArtifacts` step inside the appropriate `dir()` block.

3. **`javac` command was not available**

   * Maven failed to compile the project because only the Java Runtime Environment (JRE) was installed.

   * Since Maven requires the Java compiler (`javac`), the build could not proceed.

   * **Fix:** Installed the Java Development Kit (JDK) and verified the installation using:

     ```bash
     java -version
     javac -version
     mvn -version
     ```

# 📚 What This Pipeline Does

This pipeline performs a complete Continuous Integration (CI) workflow for a Maven-based Java application.

### Build Stage

The first stage builds the application by executing:

```bash
mvn clean package -DskipTests
```

This command performs the following actions:

* Removes any previous build artifacts from the `target/` directory.
* Compiles the Java source code.
* Downloads project dependencies if required.
* Packages the application into a JAR file.
* Skips test execution to reduce build time.

After a successful build, Maven generates the application JAR inside:

```text
target/
```

---

### Test Stage

The second stage executes:

```bash
mvn test
```

This stage:

* Compiles the test classes.
* Executes all JUnit test cases.
* Generates XML test reports using the Maven Surefire Plugin.

These reports are stored in:

```text
target/surefire-reports/
```

Jenkins later consumes these reports to display test statistics.

---

# Post Actions

The `post` block executes after all pipeline stages have finished, regardless of whether the build succeeds or fails.

It is commonly used for:

* Publishing test reports
* Archiving build artifacts
* Cleaning the workspace
* Sending notifications
* Performing cleanup operations

---

## `always`

The `always` block executes after every pipeline run, irrespective of whether the build:

* Succeeds
* Fails
* Is aborted
* Becomes unstable

This ensures that important reports and artifacts are still processed.

---

## `junit`

The following step:

```groovy
junit 'sample-apps/java-app/target/surefire-reports/*.xml'
```

Searches for all JUnit XML reports generated by Maven.

Jenkins parses these reports and displays:

* Total tests executed
* Passed tests
* Failed tests
* Skipped tests
* Test execution history
* Test trend graph across multiple builds

This makes it easy to identify failing tests directly from the Jenkins dashboard.

---

## `archiveArtifacts`

The following step:

```groovy
archiveArtifacts artifacts: 'sample-apps/java-app/target/*.jar', fingerprint: true
```

Archives the generated JAR file after the build completes.

Benefits include:

* Makes the JAR downloadable from the Jenkins build page.
* Preserves the artifact even if the workspace is cleaned later.
* Allows downstream Jenkins jobs to reuse the generated artifact.

---

## `fingerprint: true`

When fingerprinting is enabled, Jenkins calculates a unique hash for every archived artifact.

This allows Jenkins to:

* Track artifacts across multiple Jenkins jobs.
* Identify where an artifact was originally created.
* Determine which downstream jobs consumed the same artifact.
* Improve artifact traceability throughout a CI/CD pipeline.

---

# 💡 Key Learnings

- `dir()` changes the working directory only within that specific block.
- The `post` block executes from the Jenkins workspace root unless another `dir()` block is used.
- Always use the correct relative path when publishing JUnit reports or archiving artifacts.
- `mvn clean package -DskipTests` builds the application without executing tests.
- `mvn test` runs the unit tests separately and generates JUnit XML reports.
- The `junit` step publishes test results to the Jenkins dashboard.
- `archiveArtifacts` stores build artifacts permanently with the build and makes them downloadable from Jenkins.
- `fingerprint: true` helps Jenkins track artifacts across multiple builds and downstream pipelines.
